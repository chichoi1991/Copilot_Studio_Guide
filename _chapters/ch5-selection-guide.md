## Copilot Studio에 적합한 요구사항

아래와 같은 요구사항은 Copilot Studio로 **빠르고 효과적으로 구현**할 수 있는 영역입니다.

### 적합한 시나리오

| 시나리오 | 구현 방식 | 예시 |
|---|---|---|
| **사내 FAQ Agent** | Knowledge + Generative Answers | 인사, 총무, IT 정책 안내 |
| **업무 요청 접수** | Topics + Power Automate | IT 장애 신고, 비품 신청 |
| **데이터 조회** | Actions + 커넥터 | 휴가 잔여일, 매출 현황 조회 |
| **간단한 결재/승인** | Power Automate 연계 | 출장 신청, 구매 승인 |
| **안내/가이드 봇** | Knowledge + Topics | 신입 온보딩, 시스템 사용법 |
| **설문/피드백 수집** | Topics + Dataverse | 만족도 조사, 교육 평가 |

### Copilot Studio 적합 판단 기준

- ✅ 대화형 인터페이스가 자연스러운 업무
- ✅ 기존 M365/Power Platform 데이터를 활용하는 업무
- ✅ 현업 담당자가 직접 운영·수정할 수 있어야 하는 Agent
- ✅ 구축 기간이 짧아야 하는 프로젝트 (수 일 ~ 수 주)
- ✅ Teams 중심의 업무 환경

## 직접 개발(SI)이 필요한 요구사항

아래 조건에 해당하면 Copilot Studio 단독보다 **직접 개발(SI)**이 더 적합합니다.

### 직접 개발이 필요한 시나리오

| 시나리오 | 이유 | 기술 스택 예시 |
|---|---|---|
| **복잡한 멀티턴 대화** | 10+ 단계의 맥락 유지 필요 | Azure Bot Service + AOAI |
| **실시간 데이터 처리** | WebSocket, 스트리밍 필요 | Azure Functions + SignalR |
| **커스텀 UI/UX** | 채팅 외 특수 UI 요구 | React/Next.js + Bot Framework |
| **대규모 트래픽** | 수만 명 동시 접속 | AKS + Redis + 로드밸런서 |
| **레거시 시스템 통합** | SOAP, MQ, 전용 프로토콜 | Azure Integration Services |
| **온프레미스 전용** | 클라우드 접근 불가 | 자체 서버 + OSS LLM |
| **고급 RAG 파이프라인** | 커스텀 임베딩, 재순위화 | Azure AI Search + AOAI |
| **멀티 에이전트 오케스트레이션** | 다수 Agent 간 협업 로직 | Semantic Kernel / AutoGen |

### 직접 개발 선택 기준

- ❌ Copilot Studio 표준 커넥터로 연동 불가한 시스템
- ❌ 대화 흐름이 10단계를 넘어가는 복잡한 프로세스
- ❌ SLA 99.9% 이상의 고가용성 요구
- ❌ 초당 수백 건 이상의 동시 호출
- ❌ 완전한 UI 커스터마이징 필요

## 혼합 구조(Studio + 개발) 패턴

실무에서는 **Copilot Studio와 직접 개발을 결합**하는 혼합 구조가 가장 현실적인 선택인 경우가 많습니다.

### 패턴 1: Studio 프론트 + 개발 백엔드

```
┌─────────────────────────────────────────┐
│         Copilot Studio (프론트)          │
│  ┌──────────┐  ┌───────────────────┐   │
│  │  Topics   │  │  Knowledge/Gen AI │   │
│  └─────┬────┘  └───────────────────┘   │
│        ↓                                │
│  ┌──────────────────────┐               │
│  │     Actions          │               │
│  └──────────┬───────────┘               │
└─────────────┼───────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│       Azure (백엔드 개발 영역)           │
│  ┌──────────────┐  ┌──────────────────┐│
│  │Azure Functions│  │Azure AI Search  ││
│  └──────────────┘  └──────────────────┘│
│  ┌──────────────┐  ┌──────────────────┐│
│  │  SQL / Cosmos │  │  Azure OpenAI   ││
│  └──────────────┘  └──────────────────┘│
└─────────────────────────────────────────┘
```

**적합한 경우**: 대화 인터페이스는 Studio로 빠르게 구현하되, 백엔드 처리가 복잡한 경우

### 패턴 2: 개발 프론트 + Studio 오케스트레이션

```
┌─────────────────────────────────────────┐
│       커스텀 웹/앱 (프론트엔드)           │
│  React / Teams App / Mobile App        │
└──────────────┬──────────────────────────┘
               ↓ (Direct Line API)
┌─────────────────────────────────────────┐
│        Copilot Studio (오케스트레이션)    │
│  대화 관리 + Action 호출 + Knowledge    │
└──────────────┬──────────────────────────┘
               ↓
┌─────────────────────────────────────────┐
│          외부 시스템 / API               │
└─────────────────────────────────────────┘
```

**적합한 경우**: UI는 커스텀으로 만들되, 대화 로직 관리는 Studio의 비주얼 편집기를 활용하고 싶은 경우

### 패턴 3: Studio 기본 + 고급 기능 점진 개발

| 단계 | 구현 | 설명 |
|---|---|---|
| **Phase 1** | Copilot Studio 단독 | MVP: FAQ + 기본 조회 기능 |
| **Phase 2** | Studio + Power Automate | 자동화 플로우 추가 |
| **Phase 3** | Studio + Azure Functions | 복잡한 비즈니스 로직 외부화 |
| **Phase 4** | Studio + 커스텀 프론트엔드 | 필요 시 UI 커스터마이징 |

<div class="info-box tip">
<strong>💡 실무 팁</strong>
처음부터 전체 구조를 설계하기보다, <strong>Phase 1(Copilot Studio MVP)부터 시작</strong>하여 사용자 피드백을 받고, 부족한 부분만 점진적으로 개발 영역으로 확장하는 것이 가장 효율적입니다. 이를 <strong>"Studio First, Dev Later"</strong> 전략이라 합니다.
</div>
