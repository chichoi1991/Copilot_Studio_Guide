## 개인 Agent와 조직 Agent 차이

Copilot Studio에서 만든 Agent는 사용 범위와 관리 주체에 따라 **개인 Agent**와 **조직 Agent**로 구분됩니다.

### 비교 표

| 구분 | 개인 Agent | 조직 Agent |
|---|---|---|
| **사용 범위** | 제작자 + 공유한 소수 인원 | 부서 또는 전사 |
| **관리 주체** | 개인 (Maker 본인) | IT 관리팀 / CoE(Center of Excellence) |
| **환경** | 개인 환경 또는 기본 환경 | 관리형 운영 환경 |
| **거버넌스** | 낮음 (개인 판단) | 높음 (정책·프로세스 적용) |
| **Knowledge 소스** | 개인 OneDrive, 소규모 SharePoint | 전사 SharePoint, 공식 문서 |
| **유지보수** | 개인 역량에 의존 | 팀 단위 운영·유지보수 |
| **SLA** | 없음 | 조직 내부 SLA 적용 가능 |
| **예시** | 개인 업무 보조 봇 | HR Agent, IT 헬프데스크 Agent |

<div class="info-box note">
<strong>📌 핵심 차이</strong>
개인 Agent는 <strong>"내가 쓸 것"</strong>을 만드는 것이고, 조직 Agent는 <strong>"모두가 쓸 것"</strong>을 만드는 것입니다. 조직 Agent는 거버넌스, 유지보수, 데이터 관리 모두에서 더 높은 수준의 관리가 필요합니다.
</div>

## 조직 Agent 등록 프로세스

조직 공용 Agent를 배포하려면 체계적인 등록 프로세스가 필요합니다. 아래는 권장하는 단계별 프로세스입니다.

### 프로세스 흐름

```
┌─────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 1. 기획  │ → │ 2. 개발   │ → │ 3. 검증  │ → │ 4. 승인  │ → │ 5. 배포  │
│  & 신청  │   │  & 구축   │   │  & 테스트 │   │  & 등록  │   │  & 모니터 │
└─────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
```

### 단계별 상세

#### 1단계: 기획 및 신청

| 항목 | 내용 |
|---|---|
| Agent 이름 | 명확하고 직관적인 이름 |
| 대상 사용자 | 전사 / 특정 부서 / 특정 역할 |
| 핵심 기능 | Agent가 처리할 업무 범위 정의 |
| Knowledge 소스 | 연결할 데이터 소스 및 담당자 |
| 운영 담당자 | Agent 유지보수 및 콘텐츠 관리 담당 |
| 에스컬레이션 경로 | Agent가 해결 못 하는 건의 처리 경로 |

#### 2단계: 개발 및 구축

- **개발 환경**에서 Agent 구축
- Topics, Knowledge, Actions 설정
- Instructions(지침) 작성
- 내부 테스트 수행

#### 3단계: 검증 및 테스트

- **스테이징 환경**으로 이동
- 파일럿 사용자 그룹(10~30명)에게 배포
- 테스트 시나리오 기반 검증

| 테스트 항목 | 확인 내용 |
|---|---|
| **기능 테스트** | 모든 Topic이 정상 동작하는가? |
| **Knowledge 정확도** | 올바른 정보를 기반으로 응답하는가? |
| **에지 케이스** | 예상 외 입력에 적절히 대응하는가? |
| **권한 테스트** | 사용자별 접근 제어가 올바른가? |
| **성능 테스트** | 응답 속도가 적절한가? |

#### 4단계: 승인 및 등록

- IT 관리팀 / CoE 검토 및 승인
- 보안 검토 (Knowledge 소스 권한, 인증 방식)
- 조직 Agent 카탈로그에 등록

#### 5단계: 배포 및 모니터링

- **운영 환경**으로 이동
- Teams 관리센터를 통한 조직 배포
- Analytics 대시보드를 통한 사용 현황 모니터링

## 전사 공용 Agent 운영 시 유의사항

### 거버넌스

- **Agent 명명 규칙**: `[부서코드]_[Agent명]` 형식으로 통일 (예: `HR_OnboardingGuide`)
- **버전 관리**: 주요 변경 시 버전 기록을 남기고, 롤백 가능한 프로세스 구축
- **변경 관리**: Knowledge 소스 변경, Topic 수정 등은 변경 요청서를 통해 관리

### 콘텐츠 관리

- **정기 검토**: 최소 **분기 1회** Knowledge 소스의 최신성을 검토
- **피드백 수집**: Agent에 "피드백 보내기" Topic을 포함하여 사용자 의견 수집
- **FAQ 업데이트**: Analytics에서 자주 묻는 질문을 분석하여 Topic/Knowledge에 반영

### 운영 모니터링

| 모니터링 항목 | 도구 | 주기 |
|---|---|---|
| **사용량 (세션 수, 사용자 수)** | Copilot Studio Analytics | 주 1회 |
| **응답 정확도** | 사용자 피드백 + 수동 샘플링 | 월 1회 |
| **에스컬레이션 비율** | Analytics + 헬프데스크 연동 | 주 1회 |
| **에러/실패율** | Analytics | 일 1회 |

<div class="info-box warning">
<strong>⚠️ 주의</strong>
조직 공용 Agent의 Knowledge 소스가 <strong>비공개 문서나 민감 정보</strong>를 포함하는 경우, 반드시 <strong>접근 권한 감사(audit)</strong>를 수행하세요. Agent를 통해 의도치 않게 정보가 노출될 수 있습니다.
</div>

## 운영 사례

### (예) HR 가이드 Agent

| 항목 | 내용 |
|---|---|
| **대상** | 전 직원 |
| **주요 기능** | 인사 정책 안내, 휴가/급여 안내, 온보딩 가이드 |
| **Knowledge** | HR 정책 SharePoint, 사내 인사규정 문서 |
| **Topics** | 휴가 잔여일 조회 (ERP 연동), 증명서 발급 요청 |
| **에스컬레이션** | 인사팀 Teams 채널 알림 |
| **운영팀** | HR팀 (콘텐츠) + IT팀 (기술 운영) |

**기대 효과**:
- 인사팀 반복 문의 **60~70% 감소**
- 신입사원 온보딩 시간 단축
- 24시간 셀프서비스 가능

### (예) IT 헬프데스크 Agent

| 항목 | 내용 |
|---|---|
| **대상** | 전 직원 |
| **주요 기능** | IT 장애 신고, 비밀번호 초기화, S/W 설치 안내, FAQ |
| **Knowledge** | IT 운영 매뉴얼, 자주 묻는 질문 DB |
| **Topics** | 장애 티켓 생성 (ServiceNow 연동), 비밀번호 초기화 가이드 |
| **Actions** | ServiceNow API 호출, AD 비밀번호 리셋 플로우 |
| **에스컬레이션** | IT 헬프데스크 담당자 Teams 알림 |

**기대 효과**:
- L1 문의 **50~60% 자동 처리**
- 문의 접수 ~ 해결까지 평균 시간 단축
- IT 헬프데스크 운영 비용 절감
